# 银行投资分析

## 定位

把银行视为“受资本约束的高杠杆资产负债表”，而不是一般工商企业。以官方报告原文为主证据，沿着“口径统一 → 单季重构 → 驱动拆解 → 宏观传导 → 情景预测 → 误差复盘”形成判断。

这是 Codex-only 手写 Skill。它沉淀的是从历史文章中验证过的分析结构，不复刻作者的独特措辞、口头禅或修辞。

## 开始前

1. 运行 `date`，在输出中写明研究日期和数据截止日。
2. 明确任务：财报预测、财报点评、宏观分析、同业比较、文章写作或预测复盘。
3. 若做财报点评，获取指定报告期的完整官方财报；不得只读文章、快报或第三方摘要。若做前瞻预测，只获取截止日已经公开的上一期、上年同期、最近年报及必要附注，把目标期实际报告和事后资料封存到预测冻结后。
4. 若做历史时点回测，使用 `scripts/build_cutoff_manifest.py` 或等价机制建立按发布日期过滤的输入白名单，不枚举含未来文件的目录；独立复演时传入目标年/期和 `--independent-replay`，连目标期预测文章也先封存。同时封存目标期实际、后续点评、全样本回测与 `evidence-map.md`。宏观数据使用当时发布的版本，不用事后修订值。
5. 对“最新”、政策、利率、宏观数据和市场数据进行联网核验；优先使用官方发布。
6. 读取 [source-validation.md](references/source-validation.md)。再按任务加载对应参考：
   - 预测：[earnings-forecast.md](references/earnings-forecast.md)
   - 点评：完整读取 [earnings-review.md](references/earnings-review.md)、[earnings-review-data-dictionary.md](references/earnings-review-data-dictionary.md)、[earnings-scorecard.md](references/earnings-scorecard.md) 和 [report-visual-design.md](references/report-visual-design.md)
   - 宏观：[macro-bank-transmission.md](references/macro-bank-transmission.md)
   - 指标计算：[bank-metrics.md](references/bank-metrics.md)
   - 文章写作：[research-writing.md](references/research-writing.md)
   - 方法来源与边界：[evidence-map.md](references/evidence-map.md)

## 交付文件协议

1. 将财报预测、财报点评、宏观分析、同业比较和投研文章视为正式分析任务；必须生成文件，不得只在对话中输出全文。单个指标解释、口径确认或简短计算可直接回答，除非用户要求存档。
2. 用户指定路径时优先使用。否则，当前项目已有 `reports/` 时保存到 `reports/bank-investment-analysis/<银行或主题>/`；没有时保存到 `<workspace>/bank-investment-analysis-reports/<银行或主题>/`。
3. 主报告使用 UTF-8 Markdown，命名为 `YYYY-MM-DD_银行或主题_报告期或数据期_任务类型.md`。任务类型使用`财报预测`、`财报点评`、`宏观分析`、`同业比较`、`预测复盘`或`投研文章`。
4. 从对应 `assets/` 模板复制结构并填充完整内容；不把模板占位符留在交付文件中。报告必须记录研究日期、数据截止时间、分析口径、核心来源、计算方法和限制。
5. 不覆盖已有报告。同名文件存在时，若用户未要求更新原文，在文件名后增加 `_v2`、`_v3` 等版本号。
6. 财报点评只交付一个 Markdown 文件。使用 [earnings-review-data-template.csv](assets/earnings-review-data-template.csv) 作为内部144项指标检查清单，但不得将它复制、另存或作为附件交付。将原16个数据字段全部写入主报告末尾的“可追溯数据底稿”附录。
7. 完成后检查文件可打开、表格可读、链接有效、引用可追溯。最终对话只交付核心结论、必要限制和文件的绝对路径可点击链接，不重复粘贴报告全文。

## 强制分析顺序

### 1. 锁定口径

- 标记集团/母行、人民币/原币、期末/日均、累计/单季、同比/环比、年化/非年化。
- 阅读指标脚注和会计政策变更；发现重分类、并表变化、资本新规或披露公式变化时，先重述可比口径再计算。
- 把保险业务的其他业务收入与对应其他业务成本净额观察；把投资收益、公允价值变动及其他综合收益联动观察。
- 区分“披露净息差”“可比净息差”和“用净利息收入反推的净息差”，不得混用。

### 2. 重构单季

- 用累计本期减累计上期得到单季金额。
- 对平均余额、净息差和信用成本按期间年化，不把期末余额直接冒充日均余额。
- 至少展示本季、上季、上年同期三点；年报尽量展示八个单季。
- 运行 `scripts/bank_metrics.py` 中的确定性函数或其他精确计算工具，禁止心算关键值。

### 3. 建立利润构成与变动原因

先按“三收三支”理解经营主干：

`净利息收入 + 手续费净收入 + 其他非息收入 - 业务及管理费 - 减值损失 - 所得税 ≈ 净利润`

发布模型必须使用完整法定利润核对，另行安置税金及附加、其他资产减值、其他业务成本、营业外净额、少数股东损益和优先股影响。再向资产负债表、资产质量和资本约束下钻。不得用净利润增速代替经营质量判断。

### 4. 完整建立资产负债结构

- 财报点评先建数据底稿，再写叙述。主报告不得省略资产、贷款、负债与存款、风险、拨备和资本表。
- 资产表展示一级科目、占比、占比变化和对总增量贡献；分开期末余额与日均生息资产。
- 贷款先展示整体平均余额、利息收入和收益率，再分公司/零售/票据，下钻按揭/信用卡/消费贷/小微。产品收益率未披露时不得用整体收益率代填，也不要保留整列无意义N/A；年报/中报增加行业、区域和集中度。
- 负债分析至少包含期末结构、付息负债量价和存款细分量价。结构表分客户存款/批发融资、公司/零售、活期/定期；付息负债量价表展示客户存款、同业负债、应付债券、央行借款和计息负债合计；存款细分量价继续下钻公司活期/定期、零售活期/定期及公司/零售、活期/定期小计的平均余额、利息支出和年化成本率。目标季度未披露细分成本时，引用最近年报/中报的完整披露期并醒目标注期间，不用期末余额反推当季成本率。
- 手续费按财富管理、资产管理、托管、银行卡、结算清算和其他项目展示；其他非息按投资收益、公允价值、汇兑、其他业务收入/成本和OCI（其他综合收益）展示；费用按员工费用、其他业务费用、成本收入比和减值展示。手续费收入分项与手续费净收入不得混加。
- 所有必选项必须是数值或带原因的N/A，不得删行。

### 5. 打通不良生成—处置—拨备

- 按前瞻、流量、存量、处置、缓冲五层展示风险；按公司/零售及主要产品/行业下钻。
- 完成不良贷款余额变化及核对：期初+新生成+转入−现金清收−核销−批量转让/ABS（资产证券化）−上调−其他处置±汇率=期末。
- 完成贷款减值准备余额变化及核对：期初+贷款减值计提+收回已核销+其他净流入−核销/转出=期末。
- 季报未披露核销时，允许反推区间，再计算新生成不良代理下限；必须标记C/D级、公式、假设、区间和残差。
- 方向性结论至少需要一类关注/逾期/迁徙/阶段二或细分风险信号交叉验证。信号相反时只写“分化”，不宣布拐点。

### 6. 识别一阶与二阶驱动

- 净利息收入：平均生息资产 × 净息差；净息差再拆资产收益率、负债成本、结构和重定价时滞。
- 手续费：按银行卡、财富管理、托管、结算、投行等业务量与费率拆解，先检查监管降费和会计重分类。
- 其他非息：结合债券利率、交易资产配置和会计分类；将它与净利息收入的资产配置替代关系一起看。
- 业务及管理费：区分刚性预算、季节性、基数效应与主动控费；“营运费用”只在已明确定义包含项目时使用。
- 减值：同时估算资产质量“需要提多少”和管理层/监管“可能选择提多少”。
- 税：用实际税率校验，并解释免税债券、一次性项目和利润结构造成的变化。
- 资本：检查 RWA（风险加权资产）增速、核心一级资本充足率、RORWA（风险加权资产收益率）、分红及再融资约束。

### 7. 把宏观映射到账户

不要写“降息利好银行”一类单跳结论。依次回答：

`政策/数据变化 → 市场利率与信用供需 → 资产/负债重定价及规模 → 收入与信用成本 → 资本和估值`

明确即时、1–2个季度、1年以上的不同方向和时滞，并说明大行、股份行、城商行/农商行及零售/对公银行为何反应不同。

### 8. 形成评分、前瞻与可证伪条件

- 财报预测和宏观预测至少给悲观、中性、乐观三个情景；纯财报点评可只给主判断、风险情景和验证指标，除非用户要求预测。
- 多情景时按最终归母净利润或银行影响排序：悲观情景的收入通常不高于中性、成本和减值通常不低于中性；若出现反向关系，必须说明税盾、会计分类或非线性原因。
- 让情景联动：例如悲观情景不得同时假定资产收益率大降、信用风险上升，却无依据地假定减值下降。
- 预测区间用当时可得的历史误差校准；减值、交易收入或政策突变占主导时主动放宽区间。
- 对区间执行机械校验：下限 ≤ 中性值 ≤ 上限，符号、单位和百分点/百分比一致。
- 列出下一份财报前可观察的验证指标。
- 财报点评按 [earnings-scorecard.md](references/earnings-scorecard.md) 给出试行基本面评分。先用自然语言解释评分用途、发布状态、证据充分度、主要加分项、主要扣分项和最大限制，再展示六维与26个子项。C/D/N/A加权超过20%时只给暂定区间；N/A超过50%或区间宽度超过40分时暂不发布评分。即使不发布分数，也必须解释原因和已有正负证据。评分不是买卖建议。

## 分任务输出

### 财报预测

使用 [forecast-template.md](assets/forecast-template.md) 生成并保存主报告。必须包含数据截止日、输入财报、单季基线、三收三支模型、三情景、敏感性、最大误差源和可证伪条件。不要把高概率区间表述成承诺。

### 财报点评

先使用 [earnings-review-data-template.csv](assets/earnings-review-data-template.csv) 在内存或临时过程中逐项完成底稿和核对，再使用 [earnings-review-template.md](assets/earnings-review-template.md) 生成并仅保存单一主报告。将完成的内部数据通过 `scripts/render_data_appendix.py --input <临时CSV>` 校验并渲染为144行单表附录；随后不保留独立CSV。不得省略资产、贷款、负债、风险五层、不良贷款余额变化及核对、贷款减值准备余额变化及核对、RWA（风险加权资产）与资本表。比较上年同期、上一季度和可比同业；此前预测与实际的比较只在“预测复盘”任务中生成。资料不存在时保留N/A和原因。把经营贡献与少提减值、低税率、交易收益分开。最后给出试行评分、数据可靠程度、亮点、问题和证伪条件。

表格遵循紧凑契约：正文主表不超过8列，单位放在表上方，将QoQ（环比）/YoY（同比）、余额/占比、不良余额/率和数据性质/可靠程度/证据位置合并展示；长句移到表后注释。行业通用英文指标保留，但在所有读者可见位置每次都使用“英文指标（中文含义）”，不得只在首次出现时解释。内部CSV字段和计算函数名可保留英文。用户可见标题不使用未解释的“桥”。

评分模块先回答“评分衡量什么、是否发布、证据是否充分、主要加减分项是什么”，再展示计算明细。底稿附录先解释其用途和阅读方法；读者可见字段使用“数据性质、可靠程度、证据文件与页码、计算方法、估算假设、核对差额、使用限制”等自然中文，不展示裸露的 `status:`、`confidence:`、`source_file:` 等内部字段名。保持D001—D144各出现一次和原16字段可恢复。

在同一Markdown内使用Mermaid图展示重要占比，不生成外部图片附件。财报点评默认考虑资产结构、贷款结构、存款期限结构和新生不良结构四张图；只有三项以上可靠数据、分母明确且占比完成机械核对时才绘制，两项结构可在业务含义重要且合计严格等于100%时例外。每张图使用 `base` 主题和 [report-visual-design.md](references/report-visual-design.md) 的深蓝—青绿—金色投研配色，不得交付黑白默认主题；风险图固定使用橙色/红色表达风险集中。图表标题直接写结论，图下注明期间、单位、范围、口径和来源；图表不得替代表格。

内部D001—D144保持固定核心指标。若官方报告披露贷款量价、付息负债量价、手续费、其他非息或费用的更细分项目，在临时底稿末尾追加E001起的扩展记录，并由渲染器内嵌到同一附录。扩展记录使用相同16字段和证据规则；未披露时不生成，不为凑数量填N/A。

### 宏观分析

使用 [macro-analysis-template.md](assets/macro-analysis-template.md) 生成并保存主报告。先处理季节性、基数和口径，再判断趋势；最后落到银行规模、息差、手续费、信用成本、资本与估值，并列领先指标。

### 银行投研文章

先完成分析，再使用 [article-template.md](assets/article-template.md) 写作并保存主文件。标题承载核心矛盾，正文遵循“结论—证据—机制—反例—前瞻”。保留数据、假设和不确定性，不模仿语料作者的独特表达。

## 质量闸门

发布前逐项通过：

- [ ] 指定银行与报告期的官方财报已完整阅读，页码可追溯。
- [ ] 每个关键数值至少有原始来源；核心结论由第二来源或财务勾稽交叉验证。
- [ ] 累计/单季、同比/环比、年化/非年化、集团/母行均已标注。
- [ ] 前瞻预测和历史回测已按截止日封存目标期实际、后续点评、未来报告和全样本结果。
- [ ] 利润表与资产负债表驱动能相互解释；分项与合计完成勾稽。
- [ ] 财报点评的资产、贷款、负债、风险、拨备和资本强制表全部存在；每个必选单元格为数值或带原因的N/A。
- [ ] 贷款整体量价与产品结构分开；付息负债和存款细分的平均余额、利息支出、成本率及披露期间完整展示；手续费、其他非息、费用和减值完成分项核对。
- [ ] 不良贷款和贷款减值准备的余额变化核对已展示分项、残差和可信度；C/D级估算有公式、假设和区间。
- [ ] 正文主表不超过8列；读者可见英文指标每次出现都有中文含义；用户可见标题无未解释的“桥”。
- [ ] 评分模块解释评分用途、发布状态、证据充分度和主要加减分项；不发布分数时仍说明原因和已有证据。
- [ ] Markdown附录先解释阅读方法，读者可见字段全部中文化；单张144行紧凑表完整覆盖内部144项指标和16个字段，D001—D144各出现一次；交付目录中没有额外CSV底稿。
- [ ] 有细分披露时使用E编号扩展记录；D核心与E扩展指标均唯一、可恢复、可回指官方页码。
- [ ] 占比图只使用可靠数据且与表格分母核对；图表采用统一彩色投研主题和语义配色，标题、图例、图注协调；所有图内嵌Markdown，不产生外部图片附件。
- [ ] 重分类、并表、保险业务、交易资产和资本规则差异已处理。
- [ ] 事实、计算、假设和推断使用不同标签。
- [ ] 预测任务或使用多情景时，悲观/中性/乐观情景内部一致，区间算术已校验。
- [ ] 明确最大不确定项、替代解释和会使结论失效的证据。

## 禁止事项

- 不用普通企业的经营现金流/净利润比直接评价银行利润质量。
- 不把规模扩张本身视为增长质量；同时检查负债、风险权重和资本消耗。
- 不把净息差变动全部归因于 LPR；检查资产结构、存款期限、同业负债与重定价批次。
- 不用不良率单一指标评价资产质量；结合关注率、逾期、迁徙率、新生成不良、核销和拨备。
- 不用“整体稳定”、“存在较强处置”、“负债结构优化”等句子代替具体数据和勾稽。
- 不把反推核销或新生成不良代理下限写成披露事实；不在D级估算缺少交叉证据时宣布拐点。
- 不把其他非息收入单季波动线性外推。
- 不从图片缺失的历史文章中猜表格数值；回到官方报告或明确标注不可得。
- 不用宏观同比数的一个月变化直接宣布拐点；检查环比、基数、季节性和分项。
