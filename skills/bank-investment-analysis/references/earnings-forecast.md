# 银行财报预测方法

## 1. 先定义预测对象

写清银行、报告期、预测发布日期、信息截止日、目标指标及会计口径。预测年度累计结果时，先预测尚未披露的单季，再与已披露累计数相加。

保留“先验”：完成预测前不要读取目标期实际报告或事后点评。若做历史回测，先用 `scripts/build_cutoff_manifest.py` 生成 eligible/sealed 清单，使用文件时间和发布日期证明信息隔离，并只用截止日前已经形成的预测误差记录校准区间。`--report-map` 必须传含 `announcement_date` 的 resolved 映射；宏观 vintage、外部底稿等无法自动发现的资料用重复的 `--extra-sealed` 显式封存。需要隔离同日资料时，`--cutoff` 传带时区的 ISO 时间戳；只有日期时表示截至当日结束。全样本回测结论只能用于今天之后的新预测或方法评估，不能倒灌到历史时点预测。

## 2. 建立基线表

至少整理八个单季或三个年度：

| 单季 | 平均生息资产 | 净息差 | 净利息收入 | 手续费 | 其他非息 | 营运费用 | 减值 | 实际税率 | 归母净利润 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|

另列期末贷款、存款、不良、关注/逾期、贷款减值准备、RWA 和核心一级资本充足率。标记会计重分类、并表变化、监管冲击和一次性项目。

## 3. 两层利润构成与核对

“三收三支”用于理解经营主干，但发布模型使用完整法定利润核对关系：

`营业收入 = NII + 手续费净收入 + 其他非息收入`

`税前利润 = 营业收入 − 税金及附加 − 业务及管理费 − 信用减值 − 其他资产减值 − 其他业务成本 + 营业外净额`

`净利润 = 税前利润 − 所得税`

`归母净利润 = 净利润 − 少数股东损益`

少数股东损益为负时，归母净利润可以高于集团净利润。最后再按报告口径处理优先股股息、永续债利息或“归属于普通股股东”的差异。把“业务及管理费”作为明确项目；若使用“营运费用及其他”，必须列出包含税金、其他业务成本等哪些项目。

### 3.1 净利息收入

分两步：平均生息资产和净息差。

**规模**

1. 从管理层全年贷款/存款计划、现有同比和环比速度出发；
2. 结合 M2、社融和银行类型校准；
3. 检查存款吸收、资本充足率、RWA、资产荒和季末冲量约束；
4. 估计生息资产占总资产比例，而非直接用总资产增速。

历史文章的有效启发是：大行常是信用投放的政策执行器，流动性扩张时规模可能跑赢 M2，收紧时可能更快放缓；招商银行历史上曾给出“M2+2%”的负债增长目标。但这些关系只能作为银行特定时期的经验锚，必须用最新管理层计划和实际数据重估。

**净息差**

按资产端与负债端建立重定价表：

| 项目 | 余额占比 | 利率冲击 | 重定价比例 | 生效期 | NIM贡献 |
|---|---:|---:|---:|---|---:|
| 浮息对公贷款 | | | | | |
| 按揭 | | | | | |
| 消费/信用卡 | | | | | |
| 债券与同业资产 | | | | | |
| 活期/定期存款 | | | | | |
| 同业负债/存单 | | | | | |

贷款通常比存款更快重定价；存款降息只有到期滚存部分逐步生效。降息周期因此可能先压息差、后由负债成本下降缓冲。存量按揭调整、存款挂牌利率、同业存单成本和资产结构变化要分别估计。

当银行披露不足时，可用净利息收入与累计 NIM 反推平均生息资产，再重构单季 NIM。必须说明该方法对日均余额近似和口径一致性的依赖。

累计平均余额不能直接相减。若本期累计覆盖 `m` 个月、上期累计覆盖 `p` 个月：

`新增期间平均余额 = (本期累计平均余额×m − 上期累计平均余额×p) / (m−p)`

再用 `新增期间NII = 本期累计NII − 上期累计NII`，计算：

`新增期间年化NIM = 新增期间NII×12/(m−p) / 新增期间平均生息资产`

调用 `bank_metrics.incremental_average_from_cumulative` 和 `infer_annualized_nim`，并用银行披露的分季度 NII 交叉验证。

### 3.2 手续费净收入

优先使用业务驱动：

- 银行卡：交易额、贷款余额、境内外消费与费率；
- 财富管理：AUM、风险偏好、基金/保险销售和降费；
- 托管与资管：规模、费率和产品结构；
- 结算/投行/顾问：经济活动、融资节奏和监管规则。

信息不足时，用同季度历史季节性而非简单连续环比。排除疫情、监管降费、理财整改、手续费重分类等结构断点。

### 3.3 其他非息收入

把投资收益、公允价值变动、汇兑损益和其他综合收益联动观察。主要输入：

- 债券收益率的方向和变动幅度；
- 交易性与其他债权投资的配置；
- 当季卖券兑现行为；
- 权益市场、汇率及保险子公司。

净利息收入和其他非息可能因资产在生息资产与交易资产之间迁移而互补。不要把单项误差误判成总收入模型失效，也不要假设两者总能完全对冲。

有持仓分类和久期时，可把利率敏感性作为边界：FVTPL 债券的近似价格变化为 `−修正久期×收益率变化×头寸`，FVOCI 先影响 OCI、出售时才可能转入损益，摊余成本资产通常不因市价变动直接进入损益。缺乏久期、头寸或对冲信息时，不给伪精确点估计，使用宽情景并说明会计分类。

### 3.4 业务及管理费与其他经营支出

业务及管理费通常比交易收入和减值更有计划性。以预算、前三季趋势、同季季节性为主，调整：员工/科技投入、社保与监管费基数、网点与营销、年末采购、主动控费。税金及附加、其他业务成本和其他资产减值分别建行，不能混入费用后又在法定核对关系中重复扣除。

净息差承压时，管理层可能压低费用增速，但费用刚性决定其不能无限下降。

### 3.5 减值损失

这是净利润预测的首要残差，必须建“双模型”：

**资产质量需求模型**

- 新生成不良、迁徙率、关注/逾期；
- 宏观变化率、地产/零售/小微等暴露；
- 核销、拨备覆盖率和贷款拨备率；
- 上年同季基数。

可得数据足够时，用贷款减值准备余额变化定量约束：

`所需贷款减值 ≈ 目标期末贷款拨备 − 期初贷款拨备 + 核销/处置 − 收回已核销 + 其他转出净额`

`目标期末贷款拨备 = 目标贷款余额 × 目标贷款拨备率`

目标拨备率由不良/关注/逾期迁徙、组合结构和压力情景决定。全部信用减值还需加入债券、同业和表外资产减值；余额变化中缺少的清收、汇率或并表项必须成为区间，而非默认为零。

**管理层选择模型**

- 营收压力与利润平滑；
- 监管让利或行业利润要求；
- 历史各季度计提节奏；
- 现有拨备垫和未来补提空间。

情景的主要宽度应来自两模型的差，而不是任意给净利润“±1%”。当宏观/监管制度突变时，额外设置压力情景。

### 3.6 税与归母净利润

用最近三个单季实际税率做锚，并检查免税债券收入变化。税率可以解释利润变化，但不能为了达到目标净利润而随意设定。最后处理少数股东损益、优先股股息与归母口径。

## 4. 资本约束与增长可持续性

规模预测必须回答：存款是否支持、RWA 增长是否支持、核心资本是否支持。把总资产增速、RWA 增速和内生资本增量放在同一张表。

若总资产高速增长但 RWA 低增，检查国债/地方债、抵押、内评法参数和资本新规；若 RWA 快于资产，检查资产风险权重上升。

## 5. 情景设计

每个情景使用一组相互一致的假设：

| 驱动 | 悲观 | 中性 | 乐观 |
|---|---:|---:|---:|
| 平均生息资产 | | | |
| 单季NIM | | | |
| 手续费 | | | |
| 其他非息 | | | |
| 业务及管理费 | | | |
| 信用及其他减值 | | | |
| 实际税率 | | | |

按最终归母净利润给情景排序。机械检查除 `低≤中≤高` 外，还检查经济方向：悲观情景的收入通常更低、费用/减值通常更高；若税率、OCI转损益或少数股东损益使单项方向反常，必须写出原因。

另做单因素敏感性：NIM ±5bp、减值 ±10%、其他非息 ±20% 对净利润增速的影响。优先报告最大的敏感项。

## 6. 区间校准

按以下规则校准：

- 不把“中性值 ±1个百分点”默认称为高概率；没有预先定义覆盖目标并做样本外校准时，称为“模型区间”而不是概率区间；
- 只使用预测截止日前已完成、未含目标期的误差记录；优先做滚动样本外或留一校准；
- 分银行类型、报告期和主要误差源观察误差，不把另一家银行的区间宽度直接移植；
- 减值、交易收入占比高或存在政策断点时，按压力情景扩大；
- 分项区间经模型聚合，不在最终净利润上机械加固定宽度；
- 记录预测分项误差，下一期更新先验。

常规模型至少另列三类压力：制度/监管断点、NIM重定价幅度错估、减值或交易收入的管理层选择。全样本数字与历史案例只存放在 `evidence-map.md`；严格历史复演不得读取该文件。

## 7. 发布与复盘

发布前校验所有情景边界、符号和单位。实际财报发布后按分项核对预测误差：

`净利润误差 = NII误差 + 手续费误差 + 其他非息误差 − 费用误差 − 减值误差 − 税误差 ± 归母差异`

把误差分为：数据口径、模型参数、政策/宏观冲击、管理层选择、随机波动、算术/抄写。只对可学习误差更新方法。
