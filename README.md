# Bank Investment Analysis Skill

面向中国银行股投资研究的 Codex Skill。基于银行官方财报、附注、业绩会和宏观一手数据，完成财报预测、财报点评、同业比较、宏观传导及投研写作。

## 核心能力

- **财报深度点评**：分析营收、利润、净息差、手续费及佣金净收入、其他非息收入、费用、资产负债结构、资本充足率及业绩驱动因素。
- **资产质量拆解**：完整分析不良生成、迁徙、现金清收、核销、转让、拨备计提及贷款减值准备变化，并在披露不足时反推关键风险指标。
- **财报前瞻预测**：预测营业收入、净利润、净息差、信贷投放、资产质量和资本水平，并提供基准、乐观与审慎情景。
- **银行同业比较**：横向比较不同银行的盈利能力、资产质量、负债成本、业务结构和资本效率。
- **宏观经济传导**：分析利率变化、存款重定价、房地产风险、地方债化解等因素对银行经营的影响。
- **财报综合评分**：从盈利、成长、资产质量、负债、资本和经营趋势等维度评价财报表现。
- **投研文章写作**：生成结构完整、数据可追溯、包含紧凑表格与彩色图表的 Markdown 投资研究报告。

## 适用场景与提示词

下面的提示词可以直接复制使用，也可以补充指定的报告期、对比银行、关注指标和输出深度。

| 场景 | 适合解决的问题 | 示例提示词 |
|---|---|---|
| 财报深度点评 | 一份财报表现如何，增长来自哪里，风险是否改善 | `$bank-investment-analysis 请点评招商银行最新财报，完整分析盈利、资产负债结构、贷款与存款结构、非息收入、费用、资产质量、拨备和资本，并给出财报评分。` |
| 财报发布前预测 | 下一期收入、利润、净息差和资产质量可能如何变化 | `$bank-investment-analysis 请预测宁波银行下一季度财报，给出营业收入、净利润、净息差、贷款增速、不良率和拨备覆盖率的基准、乐观与审慎情景。` |
| 银行同业比较 | 多家银行谁的经营质量更好，差异来自哪里 | `$bank-investment-analysis 请比较招商银行、平安银行和兴业银行最新一期财报，重点分析盈利能力、资产质量、存款成本、零售业务、资本效率及相对优势。` |
| 经营趋势跟踪 | 单家银行多个报告期的改善或恶化趋势 | `$bank-investment-analysis 请跟踪江苏银行最近八个季度的经营趋势，分析营收、利润、净息差、信贷结构、不良生成、拨备和资本变化，并识别趋势拐点。` |
| 资产质量专题 | 不良风险从哪里产生，如何处置，拨备是否充足 | `$bank-investment-analysis 请专题分析浦发银行的资产质量，打通不良生成—迁徙—清收—核销—转让—拨备链条，并利用贷款减值准备变化反推核销区间和新生成不良下限。` |
| 负债成本专题 | 存款结构和付息率如何影响净息差 | `$bank-investment-analysis 请分析工商银行的负债与存款成本，拆分公司、零售、活期和定期存款的规模、占比及付息率，并判断后续净息差压力。` |
| 宏观与政策传导 | 利率、房地产或地方债政策怎样影响银行 | `$bank-investment-analysis 请分析降息、存款重定价和地方债置换对中国银行业的影响，说明对贷款收益率、存款成本、净息差、资产质量和资本的传导路径。` |
| 投研文章撰写 | 将分析结果整理成可发布的专业文章 | `$bank-investment-analysis 请基于官方一手资料撰写一篇招商银行最新财报的深度投研文章，包含核心结论、完整数据表、关键图表、风险分析、财报评分和投资观察。` |

## 报告输出

默认生成单一 Markdown 报告，将分析正文、紧凑数据表、彩色图表和可追溯数据底稿整合在同一文件中。关键数据会标注来源、口径、计算方法及可靠程度，便于复核和继续研究。

## 安装

在安装了 Codex 的环境中运行：

```bash
python3 "${CODEX_HOME:-$HOME/.codex}/skills/.system/skill-installer/scripts/install-skill-from-github.py" \
  --repo edward-xiao/bank-investment-analysis \
  --path skills/bank-investment-analysis
```

安装后，该Skill会出现在：

```text
${CODEX_HOME:-~/.codex}/skills/bank-investment-analysis
```

如果本地已经存在同名Skill，安装器会停止以保护原文件；请先将旧目录重命名备份，再重新安装。安装完成后，在下一轮任务中即可使用。
